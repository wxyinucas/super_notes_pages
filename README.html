目录

Journal Web ​

本目录是 super_notes 的本地日志网站。根目录仍然可以继续放 AI agent 协作契约、项目说明和其他非 Web 文件。

运行 ​

bash
npm install
npm run dev

默认 dev server 绑定在 127.0.0.1，避免把本地笔记站暴露到局域网。

生产构建：

bash
npm run build
npm run preview

GitHub Pages 发布 ​

GitHub Free 计划下，private repository 不能直接启用 GitHub Pages。当前采用“双仓库”发布：

  • 本仓库保持 private，用来写作和构建。
  • 另建一个 public 仓库，只保存 journal-web/.vitepress/dist 的构建产物，并从那里启用 Pages。

仓库根目录提供了 .github/workflows/deploy-pages.yml。push 到 main 后，GitHub Actions 会在 journal-web/ 下运行 npm ci 和 npm run build，再把构建产物推送到 public 发布仓库。

私有仓库需要配置：

  • Secret PAGES_DEPLOY_TOKEN：能写入 public 发布仓库的 token。
  • Variable PAGES_REPO：发布仓库，例如 wxyinucas/super_notes-pages。不填时默认是 当前仓库名-pages。
  • Variable PAGES_BRANCH：发布分支，不填时默认 main。
  • Variable VITEPRESS_BASE：可选。普通项目 Pages 通常自动使用 /${发布仓库名}/；用户站点或自定义域名可设为 /。

public 发布仓库的 Settings -> Pages 使用 Deploy from a branch，分支选 PAGES_BRANCH 对应分支，目录选 / (root)。

当前发布链路不做内容过滤，构建出的页面都可能公开。

本地 / 远端发布开关 ​

本地 npm run dev 和 npm run build 始终显示全部内容。公开发布时，workflow 会用 JOURNAL_BUILD_TARGET=public 构建，并排除标记为本地-only 的页面。

推荐在某个“大目录”的 index.md frontmatter 上加：

md
---
title: 某本书
publish: local
---

这个目录下的页面会继承该标记，不进入 public 发布仓库。也支持单篇页面写 public: false。本地模拟公开构建可以运行：

bash
npm run build:public

写作结构 ​

  • index.md：站点首页。
  • entries/：日志正文，建议继续按年份或主题拆分。
  • readings/：读书笔记、摘录和批注。
  • projects/：长期项目记录和阶段复盘。
  • snippets/：可复用写法、组件示例、短片段。
  • guides/：使用说明、主题说明和模板总览。
  • .vitepress/：VitePress 配置、主题、全局 Vue 组件。

旁注 ​

单段内容：

md
<MarginNote markdown="这里支持 **Markdown** 和 $a^2+b^2=c^2$。" />

多段内容：

md
<MarginNote label="旁注">

这里可以写多段 Markdown。

</MarginNote>

宽屏时旁注会浮到正文右侧，小屏时回到正文流里。

摘录与读书笔记 ​

摘录使用 <QuoteBlock>：

md
<QuoteBlock source="作者《书名》，章节或页码">

这里放原文摘录。

</QuoteBlock>

读书笔记优先组合 <QuoteBlock>、<SplitNote> 和 <MarginNote>，不要急着为每一种读书形态新增组件。

公式 ​

全站启用 KaTeX，支持 $...$、$$...$$、\(...\) 和 \[...\]。

已知设计点 ​

  • 当前聚合方式使用 VitePress 的导航、侧边栏、本地搜索，以及自定义右侧 sticky 目录。
  • 深层内容通过每层 index.md 做局部入口，导航和侧边栏只放稳定、高频的浅层入口。
  • VitePress 支持 <!-- @include: ... --> 形式的 Markdown include，可以用来把多个小文件在构建期拼成一个页面。
  • VitePress 1.6.4 的 audit 会提示 Vite/esbuild 开发服务器相关 advisory。这里已把 dev server 绑定到 127.0.0.1；如果未来 VitePress 2 稳定版发布，可以再升级以消除这类告警。
