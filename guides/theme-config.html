主题配置说明 ​

theme.config.toml 是本站的主题 token 配置文件。它集中管理字体、颜色、标题前缀、旁注样式和一部分页面留白。

修改主题时，优先改这个文件：

text
journal-web/theme.config.toml

不要手改生成文件：

text
journal-web/.vitepress/theme/generated-vars.css

这个 CSS 文件会在 npm run dev、npm run build、npm run preview 前自动重新生成。

工作流程 ​

常规流程：

bash
cd journal-web
npm run dev

修改 theme.config.toml 后，dev server 会在启动时生成主题变量。如果已经在运行，手动执行：

bash
npm run theme:generate

然后刷新浏览器页面。

构建时不需要额外操作：

bash
npm run build

prebuild 会先执行 npm run theme:generate。

文件关系 ​

text
theme.config.toml
        |
        v
scripts/generate-theme-css.mjs
        |
        v
.vitepress/theme/generated-vars.css
        |
        v
.vitepress/theme/style.css

theme.config.toml 负责“值”。style.css 负责“结构”：哪些选择器使用哪些变量、标题前缀如何出现、旁注在宽屏时如何浮动。

fonts ​

字体配置：

toml
[fonts]
body = 'ui-serif, Georgia, Cambria, "Times New Roman", "KaiTi", "STKaiti", "Kaiti SC", serif'
heading = 'ui-serif, Georgia, Cambria, "Times New Roman", "STXingkai", "Xingkai SC", "华文行楷", serif'
strong = '"Kaiti SC", "STKaiti", "KaiTi", ui-serif, Georgia, Cambria, "Times New Roman", serif'
emphasis = 'ui-serif, Georgia, Cambria, "Times New Roman", "KaiTi", "STKaiti", "Kaiti SC", serif'
mono = '"SFMono-Regular", Consolas, "Liberation Mono", Menlo, monospace'

这里故意把英文字体放在中文字体前面。浏览器会先用 Georgia、Cambria、Times New Roman 显示英文和数字；这些字体没有中文字形时，中文才会落到楷体或行楷。

strong 用于 Markdown 的 **加粗**，中文字体优先，浏览器会尽量使用或合成更重的楷体。emphasis 用于 Markdown 的 *斜体*，配合轻微斜体、柔和下划线和底色，作为读书标记式强调。

colors.light ​

浅色模式颜色：

toml
[colors.light]
brand1 = "#2f6f5e"
brand2 = "#3b826f"
brand3 = "#6aa58f"

h5 = "#4a685f"
strong = "#16231e"
italic = "#2a6555"
italicUnderline = "#8bbdac"
italicBackground = "#edf5f1"
marginNoteText = "#46564f"
marginNoteLabel = "#2f6f5e"
marginNoteBackground = "#f6f8f6"
marginNoteBorder = "#d8e2dc"
quoteText = "#34423d"
quoteLabel = "#2f6f5e"
quoteBackground = "#fbfcfb"
quoteBorder = "#d8e2dc"

含义：

  • brand1、brand2、brand3：VitePress 品牌色，也影响链接、强调色和标题前缀。
  • body：正文颜色。
  • h1 到 h5：各级标题颜色。
  • strong：Markdown **加粗** 的文字颜色。
  • italic：Markdown *斜体* 的文字颜色。
  • italicUnderline：Markdown *斜体* 的下划线颜色。
  • italicBackground：Markdown *斜体* 的柔和底色。
  • marginNoteText：旁注正文颜色。
  • marginNoteLabel：旁注 label 颜色。
  • marginNoteBackground：旁注背景色。
  • marginNoteBorder：旁注边框色。
  • quoteText：摘录块正文颜色。
  • quoteLabel：摘录块 label 颜色。
  • quoteBackground：摘录块背景色。
  • quoteBorder：摘录块边框色。
  • quoteAccent：摘录块左侧强调线颜色。

layout ​

布局配置：

toml
[layout]
wideLayoutBreakpoint = "1536px"
pageMaxWidth = "1680px"
contentColumnWidth = "976px"
docFontSize = "17px"
docLineHeight = "1.75"
headingPrefixWidth = "0.22rem"
headingPrefixTop = "0.35em"
headingPrefixBottom = "0.2em"
headingPaddingLeft = "1rem"
outlineSticky = "true"
outlineTop = "calc(var(--vp-nav-height) + 2rem)"
outlineMaxHeight = "calc(100vh - var(--vp-nav-height) - 4rem)"
marginNoteBreakpoint = "1536px"
marginNoteContentWidth = "688px"
marginNoteCompactWidth = "10.5rem"
marginNoteFullWidthBreakpoint = "1800px"
marginNoteWidth = "12.5rem"
marginNoteMarginRight = "-11.5rem"
marginNoteMarginLeft = "1rem"
outlineGap = "7rem"
outlineOffset = "5rem"
outlineWidth = "256px"
splitNoteContentWidth = "1120px"
splitNoteOutlineGap = "3rem"
splitNoteMainWidth = "6fr"
splitNoteAsideWidth = "4fr"
splitNoteGap = "2rem"
quoteFontSize = "0.96em"

常调项目：

  • docFontSize：正文基础字号。
  • docLineHeight：正文行高。
  • wideLayoutBreakpoint：启用大屏版心扩展的断点。
  • pageMaxWidth：VitePress 大屏版心宽度，会影响左侧导航和右侧目录离屏幕边缘的距离；实际使用时会被当前视口宽度限制。
  • contentColumnWidth：正文和内容组件所在的主列宽度，<SplitNote> 会受它影响。
  • headingPrefixWidth：h2 到 h5 左侧前缀宽度。
  • headingPaddingLeft：h2 到 h5 为前缀预留的左内距。
  • outlineSticky：右侧目录是否使用 sticky 浮动。当前为 true。
  • outlineTop：目录 sticky 时距离视口顶部的偏移。
  • outlineMaxHeight：目录 sticky 时的最大高度，超出后目录内部滚动。
  • marginNoteBreakpoint：旁注开始浮到正文右侧的屏幕宽度。
  • marginNoteContentWidth：带旁注页面的正文宽度。
  • marginNoteCompactWidth：刚进入大屏断点时的旁注宽度，用于避免旁注压到右侧目录。
  • marginNoteFullWidthBreakpoint：旁注恢复到完整宽度的屏幕宽度。
  • marginNoteWidth：旁注本身宽度。
  • outlineGap：正文列和右侧目录列之间的基础间距。
  • outlineOffset：带旁注页面中右侧目录向右偏移的距离。
  • outlineWidth：右侧目录列宽。
  • splitNoteContentWidth：<SplitNote> 页面使用的主阅读列宽；它优先级高于旁注页面的正文宽度。
  • splitNoteOutlineGap：<SplitNote> 页面中主阅读列和右侧目录之间的间距。
  • splitNoteMainWidth：<SplitNote> 主栏默认比例。
  • splitNoteAsideWidth：<SplitNote> 批注栏默认比例。
  • splitNoteGap：<SplitNote> 默认栏间距。
  • quoteFontSize：<QuoteBlock> 摘录块字号。

注意事项 ​

  • TOML 里的颜色建议使用十六进制，例如 "#2f6f5e"。
  • 带空格或逗号的字体栈要写成字符串。
  • generated-vars.css 是生成结果，不要手改。
  • 如果配置看起来没生效，先运行 npm run theme:generate，再刷新页面。
